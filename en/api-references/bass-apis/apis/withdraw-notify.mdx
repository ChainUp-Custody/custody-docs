---
title: Withdrawal Notification
api: POST /api/v1/withdraw/notify
description: "Call interface for withdrawal notification"
---

Please note:

1. The same txid, different addresses, separate notifications

2. The same txid, the same address, and the amount of notification will be cumulative

3. For the same deposte, please make sure that the request_id remains the same, otherwise it will cause repeated payments

4. After request decryption, request_id will be included


#### Decrypted Request Parameters
<ParamField query="address_from" type="string">
    from address
</ParamField>
<ParamField query="address_to" type="string" required>
    to address
</ParamField>
<ParamField query="txid" type="string" required>
    Transaction ID
</ParamField>
<ParamField query="trans_id" type="string" required>
    Withdrawal ID
</ParamField>
<ParamField query="amount" type="decimal" required>
    Amount
</ParamField>
<ParamField query="fee" type="decimal" required>
    Real miner fees
</ParamField>
<ParamField query="confirm" type="integer" required>
    Confirm the number. When the confirm passes 0, the status of the withdrawal record will be changed to “Paying”, and the withdrawal record will not be repeatedly obtained through the `/api/v1/withdraw/consume` interface
</ParamField>
<ParamField query="symbol" type="string" required>
    Coin Name
</ParamField>
<ParamField query="balance" type="decimal" required>
    Hot Wallet Balance
</ParamField>
<ParamField query="memo" type="string">
    to address memo ID
</ParamField>




#### Decrypted Response Data Parameters
None
