---
title: Batch Access Depositing Records
description: "Get the deposit records after a specific transaction ID"
---

`GET/api/v2/billing/depositList`

#### Decrypted Request Parameters
<ParamField query="time" type="long" required>
    current timestamp
</ParamField>
<ParamField query="charset" type="string" required>
    encoding format, no special case, pass parameter UTF-8
</ParamField>
<ParamField query="version" type="string" required>
    interface version number, no special case, pass parameter v2
</ParamField>

<ParamField query="ids" type="string" required>
    multiple top-up IDs separated by commas, up to 100 IDs
</ParamField>


#### Response Data Parameters
<ResponseField type="object[ ]">
    <Expandable title="object">
        <ResponseField name="id" type="integer">deposit unique id</ResponseField>
        <ResponseField name="uid" type="integer">deposit user id</ResponseField>
        <ResponseField name="symbol" type="string">currency</ResponseField>
        <ResponseField name="amount" type="string">deposit amount</ResponseField>
        <ResponseField name="created_at" type="long">creation time</ResponseField>
        <ResponseField name="updated_at" type="long">change time</ResponseField>
        <ResponseField name="txid" type="string">blockchain transaction ID</ResponseField>
        <ResponseField name="confirmations" type="integer">number of blockchain confirmations</ResponseField>
        <ResponseField name="address_from" type="string">deposit source address</ResponseField>
        <ResponseField name="address_to" type="string">depositing account address</ResponseField>
        <ResponseField name="status" type="integer">0 to be confirmed, 1 success, 2 failed, 4 pending KYT verification, 5 pending manual review (KYT risk level too high), 6 Pending manual review (KYT Deposit Circuit Breaker), 7 Transferred In. [Details refer to](/help-center/manage-joint/account-transfer/transfer#ii)</ResponseField>
    </Expandable>
</ResponseField>
