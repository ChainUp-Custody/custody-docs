---
title: Initiate Withdraw
description: "If the withdrawal address is the internal address of the ChainUp Alliance, it will not be linked; In addition, in order to avoid illegal withdrawal requests, when the wallet service receives a withdrawal request from a third-party application, it will call the `Withdrawal Reconfirmation Interface` provided by the third party."
---

`POST /api/v2/billing/withdraw`

#### Decrypted Request Parameters
<ParamField query="time" type="long" required>
    current timestamp
</ParamField>
<ParamField query="charset" type="string" required>
    encoding format, no special case, pass parameter UTF-8
</ParamField>
<ParamField query="version" type="string" required>
    interface version number, no special case, pass parameter v2
</ParamField>


<ParamField query="request_id" type="string" required>
    Request unique identification, up to 64 bits
</ParamField>
<ParamField query="from_uid" type="string" required>
    Transfer out user ID
</ParamField>
<ParamField query="to_address" type="string" required>
    Transfer in the user address, memo type, use “_” For example:eos_24545
</ParamField>
<ParamField query="amount" type="string" required>
    Withdrawal amount, including withdrawal service charge; The service charge needs to be configured in the background of the merchant
</ParamField>
<ParamField query="symbol" type="string" required>
    Withdrawal coin
</ParamField>


#### Response Data Parameters
<Expandable title="object">
    <ResponseField name="id" type="integer">Withdrawal ID</ResponseField>
    <ResponseField name="status" type="integer">0 under review by risk control, 1 in payment, 2 audit rejection, 4 failed, 5 success, 6 cancelled , 7 pending KYT verification, 8 pending manual review (KYT risk level too high). [Details refer to](/help-center/manage-joint/account-transfer/transfer#ii)</ResponseField>
</Expandable>
