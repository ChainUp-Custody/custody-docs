---
title: Transfer (Withdrawal)
api: POST /api/mpc/billing/withdraw
description: "Initiate a Transfer"
---

<Note>
The estimated transaction fee is an **optional parameter**. If not provided, the system will default to using the **current optimal miner fee** for signing transactions.
</Note>

#### Request Parameters
<ParamField query="sub_wallet_id" type="integer" required>
    Sub-wallet ID
</ParamField>
<ParamField query="symbol" type="string" required>
    Unique identifier for the coin, e.g., `USDTERC20`
</ParamField>
<ParamField query="from" type="string">
    Specify the transfer coin address
</ParamField>
<ParamField query="address_to" type="string" required>
    Transfer address
</ParamField>
<ParamField query="memo" type="string">
    transfer address memo, Memo type can be filled in if available, example: 123321
</ParamField>
<ParamField query="amount" type="string" required>
   	Transfer amount
</ParamField>
<ParamField query="request_id" type="string" required>
   	The unique identifier for transferring coins
</ParamField>
<ParamField query="remark" type="string">
    transfer remarks
</ParamField>
<ParamField query="fee_rate" type="string">
   	Fee rate for BTC series; Note: Do not pass along with other series parameters
</ParamField>
<ParamField query="size" type="string">
    Size in bytes for BTC series; Note: Do not pass along with other series parameters
</ParamField>
<ParamField query="gas_price" type="string">
    Gas recommended price for ETH series; Note: Do not pass along with other series parameters
</ParamField>
<ParamField query="gas_limit" type="string">
    Gas limit for ETH series; Note: Do not pass along with other series parameters
</ParamField>
<ParamField query="fee" type="string">
    Fee for other series such as DOT and TRX; Note: Do not pass along with other series parameters ;`except for gas_limit, e.g., ATOM``
</ParamField>
<ParamField query="sign" type="string">
   	RSA private key signature. Parameters involved in the signature: "request_id", "sub_wallet_id", "symbol", "address_to", "amount", "memo". For signature rules, refer to [`Co-Signer Transaction Signature Verification`].(/api-references/mpc-apis/co-signer/sign-verify)
</ParamField>



#### Response Data Parameters
<Expandable title="object">
    <ResponseField name="withdraw_id" type="integer">transfer id</ResponseField>
</Expandable>
