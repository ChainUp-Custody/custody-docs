## 一、开户与配置

### 1. 开户

我们与 DtcPay 采用 API 接入的合作模式。
 您需先以 DtcPay 商户身份完成开户，并获取对应的 API 凭证。完成配置后，即可在本系统中接入 DtcPay 的完整 OTC 能力，同时使用我们提供的各类特色功能，实现更高效的业务运营。

**开户：**
如果您选择 DtcPay 作为持有相关合规牌照的 OTC 服务商，首先需要在 DtcPay 平台创建账户。
您可以联系本平台客服获取 开户指引与协助，以降低对接复杂度并缩短接入时间；也可以直接联系 DtcPay 自行完成开户流程。
如果您已经拥有 DtcPay 账户，则可直接在本系统中进行账户添加与配置。

### 2. 系统配置

<Warning>
配置前请确保您已在 DtcPay 成功开户并具备相应账户权限（请使用带有生成API权限的账号）。
</Warning>

<Steps>
  <Step title="获取平台凭证信息">
    在本平台 OTC 页面点击「激活」按钮，系统将自动生成以下信息（支持直接复制）：
    <Frame>
      <img src="/images/help-center/otc/OTC002.png" />
    </Frame>
    - **IP 白名单地址**：用于在 DtcPay 后台授权本系统服务器的访问权限
  </Step>
  <Step title="在 DtcPay 后台创建 API">
    登录 DtcPay 管理后台，进入 **API 管理** 页面，点击「新建」，将上一步获取的 IP 白名单地址填写至对应输入框后创建。

    创建成功后，系统将生成以下三个关键参数：

    | 参数名 | 说明 |
    |---|---|
    | `API Key` | 客户端唯一标识 |
    | `API Secret` | 接口请求密钥 |
    | `Signature key` | 签名验证密钥 |
    <Frame>
      <img src="/images/help-center/otc/OTC003.png" />
    </Frame>   
    <Warning>
    请务必妥善保管上述参数，严禁以任何形式泄露。
    </Warning>
  </Step>
  <Step title="完成本系统配置">
    返回本系统，将生成的 `ClientId`、`SecretKey`、`Sign Key` 分别填入对应输入框，点击「创建」即可完成配置。
    <Frame>
      <img src="/images/help-center/otc/OTC004.png" />
    </Frame>    
    配置完成后，即可使用由 DtcPay 提供的完整 OTC 交易功能。
  </Step>
</Steps>

## 三、设置 OTC 安全策略

OTC 策略引擎用于构建严密且个性化的风控体系，支持配置**基础策略**与**高级策略**两类管理方案。

<Note>
完成审批策略配置后，所有触发审批的业务场景均将**强制执行**预设规则。系统从底层保障策略执行的唯一性，防止任何绕过或恶意篡改行为。
</Note>

### 基础策略
<Frame>
      <img src="/images/help-center/otc/OTC006.png" />
</Frame>  
基础策略是在启用 OTC 功能时**自动生成**的系统级风控方案，覆盖所有核心高敏感操作，确保关键变动始终处于受控状态。

**受控场景包括：**

| 类别 | 具体场景 |
|---|---|
| 资金变动 | OTC 充值、提现及交易操作 |
| 规则变更 | 编辑基础策略、高级策略或 API 配置 |
| 权限与账户 | 账户管理（法币账户信息）、白名单管理（稳定币地址） |

**审批机制说明：**

- **默认配置：** 创建者为审批人
- **灵活调整：** 可将最终审批人调整为任意 OTC 成员
- **节点管理：** 支持增减审批节点，点击「编辑」即可自定义审批路径
- **审计追溯：** 通过「审批策略记录」查看全部策略变更及审批执行详情

### 高级策略
<Frame>
      <img src="/images/help-center/otc/OTC007.png" />
</Frame>  

高级策略用于满足更精细、复杂的业务场景，其**执行优先级高于基础策略**：

- **优先匹配：** 系统优先校验是否触发高级策略
- **兜底机制：** 未命中高级策略时，自动切换至基础策略执行

**触发场景与条件配置：**

| 场景类型 | 可配置条件 |
|---|---|
| 交易与资金提现 | 可针对 OTC 交易、法币/稳定币提现，通过成员、币种、单笔限额等条件触发审批流 |
| 敏感配置管理 | 在白名单、账户管理及 API 配置场景中，可指定特定操作成员触发更高权限审批 |

<Frame>
      <img src="/images/help-center/otc/OTC008.png" />
</Frame>  
每条高级策略均支持配置**独立审批路径与节点**，实现差异化风控管理。

## 四、稳定币白名单与法币账户管理

### 稳定币地址白名单

稳定币地址白名单用于管理与限制**提现地址**，确保资金仅流向您控制的钱包地址。

<Warning>
白名单地址**必须为您掌控私钥的钱包地址**，不可填写交易所或平台托管地址。
同时，向 DtcPay 充值稳定币时，也必须从白名单地址发起，否则将无法正常入账。
</Warning>

**配置步骤：**

<Steps>
  <Step title="添加地址">
    在稳定币白名单页面点击「添加」，输入钱包地址并填写**地址别名**（建议避免相似命名，以防混淆）。
  </Step>
  <Step title="选择网络与币种">
    选择对应的区块链网络及币种后点击「确定」保存。
  </Step>
  <Step title="等待审批并启用">
    添加操作将触发安全策略审批。审批通过后地址默认为**禁用**状态，需手动开启后方可使用。
  </Step>
</Steps>
<Frame>
      <img src="/images/help-center/otc/OTC009.png" />
</Frame>  
**风控验证说明（启用地址时）：**
当您启用此地址时，系统有可能会触发风控验证，如需要请按提示完成以下流程：

1. **小额转账：** 按提示完成指定金额的小额转账
2. **核对地址：** 必须使用触发风控的地址进行转账
3. **等待激活：** 转账完成后等待系统验证

<Note>
验证资金将**全额计入**您的 DtcPay 钱包余额，验证通过后可正常使用，不会产生资金损失。
</Note>

## 五、充值

DtcPay 采用预充值模式。在进行 OTC 交易前，您需先将法币或稳定币充值至 DtcPay 指定账户，用于后续交易的资金结算与流转。

账户余额可根据业务需要灵活使用，支持随时提现至您的银行账户或区块链地址，实现资金的高效调度与管理。

### 稳定币充值
<Frame>
      <img src="/images/help-center/otc/OTC0012.png" />
</Frame>  
操作路径：
选择目标币种 → 点击【充值】→ 选择对应区块链网络 → 选择充值白名单地址

风险提示：
请务必核对充值网络信息（如 ERC20、TRC20 等）。若因网络选择错误导致资产丢失，相关风险需自行承担。

到账说明：
确认信息无误后，向系统提供的地址转入资金。资金将在链上确认后自动到账，到账后即可用于 OTC 交易。

<Warning>
请务必核对区块链网络（如 ERC20、TRC20）。因网络选择错误导致的损失需自行承担。
</Warning>

### 法币充值

<Steps>
  <Step title="创建订单">
    选择目标法币，点击【充值】并确认订单，系统将展示收款账户信息。
    <Frame>
      <img src="/images/help-center/otc/OTC0013.png" />
    </Frame> 
  </Step>
  <Step title="线下汇款">
    按页面指引完成银行转账。
    <Frame>
      <img src="/images/help-center/otc/OTC0014.png" />
    </Frame>
    <Warning>
    汇款备注中**必须填写 Reference No.**，否则可能导致无法入账或资金损失。

  </Step>
  <Step title="上传凭证">

    <Frame>

  <Step title="等待到账">
    审核通过后资金将计入余额。如长时间未到账，请联系 DtcPay 客服。
  </Step>

##  七、OTC 交易

### 1. 什么是 OTC 交易

OTC（Over-The-Counter）交易，是指用户在平台上按照实时报价，直接完成法币与数字资产之间的兑换。  
交易由OTC服务商（DTC Pay）统一撮合并完成结算，而非通过公开市场挂单撮合。

在 DTC 的 OTC 交易中，主要用于法币与稳定币或其他数字资产之间的兑换。同时，部分场景下也支持：
- 法币与法币之间兑换
- 稳定币与稳定币之间兑换

> 具体支持的交易币对，请以系统实际展示为准。

**常见交易示例：**
- 使用 USD / SGD 等法币购买 USDT
- 使用 USDT / USDC 等稳定币兑换法币

---

## 如何进行 OTC 交易

在发起 OTC 交易前，请确保用于支付的账户中余额充足。  
如余额不足，请先完成充值操作（可参考「法币充值」或「稳定币充值」指南）。

完成准备后，按照以下步骤操作：
<Frame>
      <img src="/images/help-center/otc/OTC0018.png" />
</Frame>

### 1. 选择兑换币种
选择您希望卖出的币种（支付币种），以及希望获得的币种（目标币种）。

### 2. 填写交易金额
输入本次需要兑换的金额，系统将基于当前报价，实时展示对应的兑换结果。

### 3. 确认交易
核对交易信息无误后提交申请。  
部分交易可能需要经过审批流程，审批通过后才会执行。

### 4. 汇率说明
由于报价存在有效期限制，如审批完成时原报价已失效，系统将按最新有效汇率执行兑换。

> 最终成交汇率以交易实际执行时为准。

---

交易完成后，系统将根据最终确认的汇率完成资金兑换，并自动更新您的账户余额。
